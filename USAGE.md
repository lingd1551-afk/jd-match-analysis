# 使用说明

## JD匹配度分析功能使用指南

### 功能概述

JD匹配度分析功能可以帮助您：
1. 上传岗位描述（JD）
2. 自动分析JD中的关键信息
3. 计算与简历的匹配度
4. 获得详细的优化建议

### 使用步骤

#### 1. 访问JD匹配页面

- 从首页点击"JD 匹配度分析"卡片
- 或直接访问 `/jd-match` 路由

#### 2. 输入岗位描述

在文本框中粘贴或输入完整的岗位描述，建议包含：
- 岗位名称
- 岗位职责
- 任职要求
- 技能要求
- 其他相关信息

**示例JD内容：**
```
岗位：前端开发工程师

岗位职责：
1. 负责公司产品的前端开发工作
2. 使用React、TypeScript等技术栈开发用户界面
3. 优化页面性能，提升用户体验
4. 与后端团队协作完成接口对接

任职要求：
1. 熟练掌握React、Vue等前端框架
2. 熟悉TypeScript、JavaScript
3. 有2年以上前端开发经验
4. 熟悉Git版本控制工具
```

#### 3. 开始分析

点击"开始分析"按钮，系统将：
- 提取JD中的关键信息
- 分析技能匹配度
- 计算总体匹配度
- 生成优化建议

#### 4. 查看匹配报告

分析完成后，会自动跳转到报告页面，报告包含：

**总体匹配度**
- 0-100分的匹配度评分
- 匹配度等级（高/中/低）

**优势分析**
- 已掌握的技能
- 符合要求的经验
- 其他优势点

**待提升项**
- 缺失的技能
- 需要改进的地方

**优化建议**
- 针对性的改进建议
- 技能提升方向

**简历修改建议**
- 具体的简历优化方向
- 关键词使用建议

### 报告解读

#### 匹配度评分说明

- **80-100分**：匹配度较高，建议优化细节后投递
- **60-79分**：匹配度中等，需要重点提升部分技能
- **30-59分**：匹配度较低，建议针对性提升后再投递

#### 如何根据报告优化简历

1. **突出匹配的技能**
   - 在技能部分优先展示JD中要求的技能
   - 在项目经历中详细描述相关技术的使用

2. **补充缺失的技能**
   - 如果缺少某些技能，可以在简历中添加相关学习经历或培训
   - 强调快速学习能力和相关经验

3. **使用JD中的关键词**
   - 在简历描述中使用与JD相同的关键词
   - 提高通过ATS（自动筛选系统）的概率

4. **量化工作成果**
   - 使用具体数字展示工作成果
   - 突出项目的影响和价值

### 注意事项

1. **JD内容要完整**：输入完整的岗位描述可以获得更准确的分析结果
2. **当前使用模拟数据**：系统当前使用模拟简历数据进行匹配，实际应用中应使用真实简历数据
3. **定期更新简历**：根据不同的JD要求，调整简历重点
4. **结合人工判断**：AI分析结果仅供参考，建议结合实际情况进行判断

### 技术说明

- 匹配度计算基于技能匹配（60%）和经验匹配（40%）
- 系统会自动识别常见的技术栈关键词
- 分析结果会实时生成，无需等待

### 常见问题

**Q: 为什么匹配度较低？**
A: 可能原因包括：技能不匹配、经验不足、JD要求较高。建议根据报告中的建议进行针对性提升。

**Q: 如何提高匹配度？**
A: 根据报告中的"待提升项"和"优化建议"，补充相关技能和经验，调整简历重点。

**Q: 报告数据会保存吗？**
A: 当前版本报告数据存储在浏览器sessionStorage中，刷新页面后可能丢失。建议及时保存或打印报告。
